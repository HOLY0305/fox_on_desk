# Clyde 魔改开发计划

> 最后更新：2026-05-12
> 基于 v0.1.6 源码

---

## 项目背景

在 Clyde（AI 编程桌宠）基础上进行功能扩展和体验优化。核心原则：**小体积内完成更多功能，不堆砌无用功能**。

---

## 功能规划

### 功能一：音效系统

#### 目标
关键事件发生时播放像素风格音效，提供听觉反馈。

#### 触发场景

| 事件 | 音效类型 | 触发来源 |
|------|---------|---------|
| 权限请求到来 | 短促提示音（叮） | Claude Code PermissionRequest |
| 任务完成 | 愉悦完成音 | Stop/PostCompact 事件 |
| 错误发生 | 警告音 | PostToolUseFailure 事件 |
| 新会话开始 | 轻快提示音 | SessionStart 事件 |

#### 设计约束
- 音效风格：**像素风 8-bit**，与现有 SVG 像素角色风格统一
- 音量：使用系统音量，应用内只提供**静音开关**
- 静音开关与现有 DND（勿扰）模式联动：DND 开启时自动静音
- 音效来源：免费素材库（如 freesound.org、mixkit.co 等）
- 打包体积增加控制在 **500KB 以内**（4-6 个短音效文件）

#### 技术方案
- 使用 Rust `rodio` crate 播放音频
- 音效文件打包到 Tauri resources 目录
- 在 `state_machine.rs` 的状态更新路径中插入音效触发点
- `prefs.rs` 新增 `sound_enabled: bool` 字段（默认 true）
- 托盘菜单和右键菜单添加"音效"开关选项

### 功能二：会话状态徽章

#### 目标
用户能**一眼看到**当前各会话的运行状态，通过宠物旁边的小圆点颜色区分。

#### 设计方案

**核心思路**：在宠物旁边显示一组小圆点，每个圆点代表一个活跃会话，颜色表示该会话的状态。

**颜色含义**（按会话状态）：

| 颜色 | 状态 | 含义 |
|------|------|------|
| 🟢 绿色 | idle / attention（完成） | 会话空闲或任务已完成 |
| 🔴 红色 | working / thinking | 正在执行中 |
| 🟡 黄色 | 等待审批 | 权限请求待处理 |
| 🔵 蓝色 | juggling / conducting | 子代理工作中 |
| ⚪ 灰色 | sleeping | 会话休眠 |

**显示规则**：
- 0 个活跃会话时不显示
- 1 个会话显示 1 个小圆点
- 多个会话横向排列小圆点
- 每个圆点带微弱呼吸动画（活跃状态）

**位置**：宠物窗口右上角，跟随宠物移动，mini 模式下隐藏

**实现方式**：
- 在 pet 窗口的 Svelte 层上叠加 HTML 元素（不需要额外窗口）
- 监听 state-change 事件，从 StateMachine 的 sessions 中读取各会话状态
- 圆点使用 CSS 动画 + 颜色过渡

### 功能三：键盘快捷键 — 宠物召来

#### 目标
按快捷键让宠物**瞬间移动到当前鼠标位置**。

#### 设计方案

**核心交互**：按下 `Alt+C`（C = Come），宠物从当前位置动画移动到鼠标所在位置。

**动画效果**：
- 宠物从原位置以抛物线弧度跳到鼠标位置（复用现有的 `animate_parabola` 函数）
- 落地时有轻微弹跳
- 如果宠物在 Mini 模式，先退出 Mini 再跳过来

**扩展思考**：
- 这个功能的本质是"快速召唤"，可以进一步扩展为：
  - **鼠标跟随模式**（再次按快捷键切换）：宠物缓慢跟随鼠标移动，像一个浮动的伙伴
  - **归位**（第三次按）：回到上次保存的位置
  - 三种状态循环：正常 → 跟随 → 归位

**实现方式**：
- 使用 Tauri 的 `tauri-plugin-global-shortcut`
- 获取鼠标位置后调用现有的窗口移动 + 动画逻辑
- `prefs.rs` 新增 `follow_cursor: bool` 字段

### 功能四：宠物尺寸与新动画

#### 4.1 尺寸问题

**现状**：宠物最大尺寸 L = 360x360 物理像素，在高分屏上确实不大。

**方案**：新增 XL 尺寸：**480x480** 物理像素。

**实现**：
- `prefs.rs` 的 `size_to_pixels()` 新增 "XL" → (480, 480)
- 托盘菜单和右键菜单的 Size 子菜单添加 XL 选项
- SVG 是矢量图，放大不会模糊

#### 4.2 新动画开发

**目标**：增加新动画帧，丰富宠物表现力。

**建议的新动画**：

| 动画 | 场景 | SVG 文件名 | 说明 |
|------|------|-----------|------|
| 挠头 | 遇到错误/困惑时 | `clyde-idle-scratch.svg` | 长时间无操作的自主小动作 |
| 喝咖啡 | 长时间工作后的休息 | `clyde-idle-coffee.svg` | 配合时间感知 |
| 挥手 | 新会话开始时 | `clyde-react-wave.svg` | 打招呼 |
| 鼓掌 | 任务完成时 | `clyde-react-clap.svg` | 更生动的完成反馈 |
| 派活 | 启动子代理时 | `clyde-working-assign.svg` | 大宠物给小宠物布置任务 |

**动画制作方式**：
- 保持现有像素风格
- 参考 `assets/svg/` 中现有 SVG 的结构（viewBox、元素 ID 约约）
- 每个动画帧 2-5KB
- 可以用 Aseprite 或 Piskel（免费在线像素画工具）制作，然后导出 SVG

### 功能五：新对话 — 快速选择项目目录

#### 目标
快速选择一个项目文件夹，在该目录下启动 Claude Code 终端。

#### 设计方案

**触发方式**：
- 宠物处于 idle 状态时，**双击**宠物弹出项目选择面板（替换原有的"戳一下"反应，原功能无实用价值）
- 或者通过右键菜单新增"新对话"选项

**交互流程**：
```
双击宠物 → 弹出文件夹选择对话框 → 用户选择目录 → 在该目录下打开 Claude Code 终端 → 宠物进入 working 状态
```

**实现方式**：
- 使用 Tauri 的 `tauri-plugin-dialog`（文件/文件夹选择对话框）
- 选择目录后，通过 `std::process::Command` 在该目录下启动终端 + Claude Code
- Windows: `powershell -NoExit -Command "Set-Location '{path}'; claude"`
- `prefs.rs` 新增 `recent_projects: Vec<String>` 记录最近打开的项目（最多 10 个）
- 右键菜单可显示"最近项目"子菜单快速访问

### 功能六：长任务进度气泡

#### 目标
长时间运行的任务中，通过气泡通知用户当前进度。

#### 设计方案

**触发场景**：

| 事件 | 气泡内容 | 触发来源 |
|------|---------|---------|
| 子代理完成 | "子代理完成：{摘要}" | SubagentStop 事件 |

**显示规则**：
- 仅在子代理完成时显示，3 秒后自动消失
- 复用现有的权限气泡窗口系统（`permission.rs` 的 BubbleMap）
- 新增 `WindowKind::ProgressNotice` 类型
- 气泡样式与 ModeNotice 一致（蓝色主题）
- DND 模式下不显示进度气泡

**气泡内容**：
```
[Clyde] 子代理完成
摘要：已修复 auth 模块的 3 个测试用例
3 秒后自动消失
```

**实现方式**：
- `permission.rs` 新增 `show_progress_bubble()` 函数
- `state_machine.rs` 的 `update_session_state()` 中检测 SubagentStart/SubagentStop 事件
- 使用现有的 `prepare_close_bubble` + tokio 定时器实现自动消失
- `prefs.rs` 新增 `progress_bubbles_enabled: bool`（默认 true）

## 已取消的功能

以下功能经讨论后取消，记录原因供后续参考：

| 功能 | 取消原因 |
|------|---------|
| 拖拽内容到宠物 | 实用价值不大，增加复杂度 |
| 简单对话能力 | 不接入 AI 的关键词匹配太弱，接入 AI 则增加体积 |
| 全局快捷键（DND/Mini/隐藏） | 使用频率低，不值得增加依赖 |

## 技术优先级

按投入产出比排序：

| 优先级 | 功能 | 理由 |
|--------|------|------|
| 1 | 音效系统 | 独立模块，不影响现有架构，效果立竿见影 |
| 2 | 会话状态徽章 | 纯前端叠加，逻辑简单，实用价值高 |
| 3 | 宠物召来快捷键 | 核心逻辑 3 行代码，有趣且实用 |
| 4 | 新对话目录选择 | 实用功能，Tauri 插件直接支持 |
| 5 | 长任务进度气泡 | 复用现有气泡系统，增量开发 |
| 6 | 尺寸调整 | 改配置值，简单但需测试 DPI |
| 7 | 新动画 | 美术工作，与代码开发可并行 |

## 开发备注

### 关于 "Permission denied by hook" 报错
这是 Claude Code 自身的工具权限 hook 机制导致的，不是 Clyde 代码的问题。当 Claude Code 的 settings.json 中配置了权限 hook，某些工具调用会被 hook 拦截。Clyde 注册的 hook 只监听事件（SessionStart、PermissionRequest 等），不会拦截工具调用。此报错可忽略。
